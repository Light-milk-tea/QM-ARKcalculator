# 明日方舟 DPS 计算器

## 1. 背景与目标

在游戏《明日方舟》中，角色的总伤和 DPS（每秒伤害）是评定输出能力的重要参数。但由于：

- 伤害计算乘区复杂（加算/乘算、攻击间隔/攻速等交互多）
- 游戏官方技能描述存在省略与模糊（同一描述在不同技能中可能语义不一致）
- 天赋、模组、分支职业的“隐藏规则”与覆盖逻辑较多

因此，本项目的目标是提供一款对普通玩家、攻略作者都友好的工具，能够在同一页面里计算并展示：

- 干员基础信息与面板：生命、攻击、防御、法抗、攻击间隔、伤害类型、攻击速度等
- 计算结果：单次伤害（DPH）、总伤、DPS，以及计算过程的可解释报告

### 2.1 影响计算的核心 JSON（`public/json/`）

- `character_table.json`：干员主数据（基础面板、成长、天赋引用、潜能、信赖、技能引用等）
- `skill_table.json`：技能主数据（各等级参数与 blackboard）
- `uniequip_table.json`：模组元数据（归属、解锁、材料等）
- `battle_equip_table.json`：模组战斗数据（面板增益、特性/天赋覆盖的 blackboard）
- `profession.json`：主职业枚举字典（大类）
- `subProfessionId.json`：分支职业字典（细分分支；含攻击类型/站位等战斗语义）

## 3. 关键术语与概念

- 面板（Final Stats）：计算时真正参与公式的最终属性（攻击/防御/法抗/攻击间隔/攻速等）
- DPH（Damage Per Hit）：单次攻击的伤害（通常指“主伤害流”的一次结算，也可能包含多段/追加流）
- DPS（Damage Per Second）：单位时间伤害。通常为“总伤 / 有效持续时间”
- Blackboard：技能、天赋、模组等效果的参数表；同一 key 在不同上下文可能需要额外语义解释
- 规则补丁（Custom Rules）：对“官方数据难以表达的语义/特例机制”进行显式编码补齐，并产生命中记录（Rule Trace）

## 4. 核心计算过程

### 4.1 第一步：构建最终面板

面板由多来源叠加而成，主要包括：

- 角色基础数值与成长曲线（等级、精英阶段）
- 信赖加成
- 潜能加成
- 模组增益（面板增益、特性/天赋覆盖）
- 天赋效果（常驻、条件触发、覆盖/替换）
- 分支职业规则（如攻击类型、站位、特殊攻击逻辑等）

<br />

示例：

```
面板攻击力 = 基础攻击力
           * (1 + 技能提供的 atk% 加成 + 天赋提供的 atk% 加成 + 模组/分支提供的 atk% 加成)
           + 各类 flat 攻击力加成（若存在）
```

### 4.2 第二步：解析技能与效果参数

技能的每个等级都有一份参数（blackboard）。计算需要将其归一化为统一的效果结构，例如：

- 攻击力倍率、攻击间隔变化、攻速变化
- 伤害类型变化（物理/法术/真实等）
- 追加伤害与多段逻辑
- 特殊语义（“下一次攻击”“每 X 次攻击”“对某类敌人”等）通过 Custom Rules 补齐

### 4.3 第三步：生成攻击排程（时间轴）

DPS 的关键不是“单次伤害”，而是“单位时间内能打多少次”。排程需要考虑：

- 基础攻击间隔与攻速换算
- 技能持续时间、回转方式（自动/手动、自然回转/攻击回转等）
- 技能期间攻击模式变化（如多段、停手、额外攻击、持续伤害 tick 等）

<br />

### 4.4 第四步：逐次结算单次伤害（DPH）

每次结算都会走统一公式层，典型输入包含：

- 进攻侧：最终面板、倍率、穿甲/易伤、伤害类型等
- 防守侧：目标防御/法抗、减伤等（由上下文提供）

输出是单次伤害值，并附带关键中间量，便于审计与定位差异。

### 4.5 第五步：汇总总伤与 DPS

最终结果通常包含：

- 总伤（各伤害流累加）
- DPS（总伤 / 有效持续时间）
