# 阶段A（0-2个月）周度拆解（MVP稳定化）

## 目标范围

- 目标：将当前骨架推进为“可稳定使用”的单目标 DPS MVP。
- 覆盖能力：筛选、参数输入、结果计算、可解释报告、基础质量门禁。
- 非目标：HPS、召唤物独立建模、元素混算、多目标模拟。

## 第1周：核心协议冻结

- 交付项：
  - 冻结 `CalculationInput` / `CalculationResult` / `RuleTrace` 字段协议。
  - 明确 warnings 最小分类（未适配、歧义、人工复核）。
  - 建立 `calc-core` 入口和最小样例。
- 验收标准：
  - `packages/calc-core` 类型检查通过。
  - 协议文档在 `docs` 可检索，字段命名不再频繁变更。

## 第2周：数据与索引稳定

- 交付项：
  - 六个核心 JSON 接入与同步脚本稳定化。
  - raw 数据适配到 `OperatorIndex` 的规则收敛。
  - 关键异常数据具备明确 warning 输出。
- 验收标准：
  - `npm run sync:data` 可稳定执行。
  - 选定样例干员可成功构建索引并触发计算。

## 第3周：计算流水线闭环

- 交付项：
  - 五步流水线（上下文、规则归一化、排程、伤害、汇总）完整可运行。
  - 覆盖物理/法术/切换态三类代表样例。
  - 报告中包含 summary/schedule/streams/ruleTrace/warnings。
- 验收标准：
  - 最小回归集通过（至少3条核心样例）。
  - 结果全为有限数值（非 NaN/Infinity）。

## 第4周：前端MVP页面可用

- 交付项：
  - 干员筛选、技能选择、敌方参数输入、结果展示联通。
  - localStorage 持久化参数。
  - 可解释区域展示 ruleTrace 与 warnings。
- 验收标准：
  - 页面可完成“选择 -> 计算 -> 查看解释”的完整流程。
  - 主要参数刷新后仍保留。

## 第5周：规则迁移第一批

- 交付项：
  - 第一批 custom 规则（伤害类型切换、破甲/减抗、额外流、短爆发修正）迁移。
  - Top20 候选清单初版（优先高频与高争议技能）。
  - 规则审计脚本输出可读统计。
- 验收标准：
  - 第一批规则均有回归样例。
  - 审计输出能定位“疑似需 custom”的技能规模。

## 第6周：质量门禁与发布准备

- 交付项：
  - 统一质量门禁：typecheck + unit + regression + build + audit。
  - CI 工作流初版与失败定位说明。
  - 版本化与回滚策略文档初版。
- 验收标准：
  - 本地一键执行质量门禁成功。
  - CI 可在 PR 上复现同等门禁结果。

## 每周固定例行

- 周一：迁移计划确认（本周规则与样例）。
- 周三：中期检查（阻塞、数值偏差、风险更新）。
- 周五：回归报告与下周计划冻结。

## 阶段A退出条件（DoD）

- MVP主流程可稳定使用并具备可解释输出。
- 至少一套黄金样例回归集稳定通过。
- 有明确的 warnings 与规则审计路径。
- 质量门禁具备本地与 CI 双轨保障。
