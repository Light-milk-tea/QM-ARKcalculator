# UI 优化与 Arkrec Wiki 风格深度还原计划

## 1. 当前状态分析

目前 `HomePage.vue` 虽然已经切换到了白底和浅色表格，但布局仍是传统的“左侧表单 + 右侧结果”的后台管理/控制台风格，缺乏纯正的 Wiki 词条排版质感。Arkrec Wiki (基于 MediaWiki) 的典型特征包括：

* 经典的“右侧 Infobox (信息框)”用于展示实体（干员）属性。

* 左侧作为主正文流，通过标准的二级带下划线标题 (`== 标题 ==`) 划分内容区块。

* 特定的标准色：边框灰 (`#a2a9b1`)、表头背景 (`#eaecf0`)、链接蓝 (`#0645ad`)。

## 2. 拟定修改内容

### 2.1 更新 Tailwind 配置 (`tailwind.config.js`)

* 调整 `arkrec` 调色板，严格对齐 MediaWiki 标准色系：

  * `border`: `#a2a9b1`

  * `panel`: `#f8f9fa`

  * `header`: `#eaecf0`

  * `link`: `#0645ad`

### 2.2 优化全局样式 (`src/style.css`)

* **Wiki Header (`.wiki-header`,** **`.wiki-subheader`)**：统一底部边框为 `#a2a9b1`，调整字号和字体粗细，还原 Wiki 词条标题样式。

* **Wiki Table (`.wiki-table`)**：还原 `wikitable` 经典样式，表头背景设为 `#eaecf0`，内边距和字体颜色更紧凑。

* **Infobox (`.wiki-infobox`)**：新增信息框样式，固定宽度，内部表格去边框化或采用极简线框，用于右侧属性展示。

* **表单控件**：弱化表单控件的现代感，采用更基础、朴素的输入框样式。

### 2.3 重构 Vue 组件 (`src/pages/HomePage.vue`)

* **布局倒置与流式排版**：

  * **右侧区域**：将干员“基础属性”抽取为右侧浮动的 `Infobox`。

  * **左侧正文**：依次垂直排列“参数设置”、“计算结果”、“规则追踪 (Rule Trace)”、“系统警告 (Warnings)”。

* **表单整合**：将“选择干员”、“选择技能”和“敌方参数设置”整合到正文区的参数配置表格或面板中，使之像维基百科的“查询计算”工具。

* **顶部导航**：弱化为纯白极简导航，去除所有非必要的投影，保留最基础的面包屑/站点标题感。

## 3. 假设与决策

* **决策**：舍弃现有的左右等宽或左窄右宽的控制台布局，转而使用典型的 Wiki 文章流布局（右侧浮动 300px Infobox，左侧自适应正文）。

* **假设**：用户希望获得沉浸式的“查维基资料”体验，而非“使用现代 SaaS 软件”的体验。

## 4. 验证步骤

1. 实施上述代码修改。
2. 通过 `vite` 预览界面。
3. 检查各区块标题、表格、输入框是否具备纯正的 MediaWiki 视觉特征。
